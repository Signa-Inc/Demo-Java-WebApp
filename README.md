# Demo-Java-WebApp

Simple CI/CD demo with Jenkins, Nexus, Tomcat



###### Как работает CI CD?



Берёт код → собирает → тестирует → сохраняет в артефакторий → развёртывает → проверяет, что всё работает.

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

|               Компонент               |        Назначение        |

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

|                 GitHub                | Хранилище исходного кода |

|                Jenkins                |   Система автоматизации  |

|                 Maven                 |    Сборка Java-проекта   |

| JUnit + RestAssured + Embedded Tomcat |       Тестирование       |

|                 Nexus                 |   Хранилище артефактов   |

|                 Tomcat                |     Сервер приложений    |

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~





###### Как работает интегрированное тестирование?



Тест внутри себя:

1. Запускает мини-Tomcat (встроенный сервер, который работает прямо в памяти Java-процесса, без отдельного запуска Tomcat).

2\. Регистрирует твой HelloServlet в этом мини-Tomcat по пути /hello.

3\. Отправляет HTTP-запрос к http://localhost:СЛУЧАЙНЫЙ\_ПОРТ/webapp-demo/hello.

4\. Проверяет, что:

&nbsp;  Сервер ответил статусом 200 (то есть «всё нормально»),

&nbsp;  В теле ответа есть фраза Hello from CI/CD Pipeline!.

Если всё это сработало — тест зелёный (успешный).

Если что-то пошло не так (например, сервлет не запустился, или текст не тот) — тест падает, и Jenkins не деплоит приложение в продакшен.
